## 🎯 Краткое описание каждой фазы

| # | Название | Что делается | Результат |
|---|----------|--------------|-----------|
| 1 | Баланс постов | Добавить поле баланса в JSON, UI меню | Пользователь видит баланс (30 бесплатных) |
| 2 | Stripe | Реальные платежи через Stripe webhook | Оплата работает, баланс обновляется |
| 3 | Проверка баланса | Проверить баланс перед рассылкой/тестом | Без баланса = рассылка невозможна, тест бесплатный |
| 4 | Списание | Вычитать только за успешные публикации | Посты тратятся только за отправленные |
| 5 | Безопасность тестов | Отправка → проверка → удаление тестовых сообщений | Тесты безопасны, группы чистые |
| 6 | Уведомления | Уведомление о низком балансе (1 раз в день) | Пользователь знает когда заканчивается баланс |
| 7 | Защита | Cooldown 30 сек, лимит 5 тестов/день | Защита от спама |

## 💡 Ключевые концепции

### Баланс
- По-умолчанию: 30 бесплатных постов при /start
- Покупаются через Stripe (4 тарифа)
- Хранится в `broadcast_state.json["balance"]`

### Списание
- Только за успешные публикации (sent_count)
- Если 7 из 10 групп → потратил 7 постов
- История в JSON

### Тесты
- **Бесплатные** (не тратят баланс)
- Отправляют сообщение с меткой 🧪
- Через 60 сек проверяют (сообщение еще в группе?)
- Потом удаляют сообщения (чистая группа)
- Защита: cooldown 30 сек, макс 5 в день

### Уведомления
- О низком балансе (≤30 постов, по-умолчанию)
- Один раз в день
- Пользователь может изменить порог

## ⚠️ Важные моменты

1. **OWNER_ID** — разработчик, может тестировать и рассылать бесплатно
2. **Обычные пользователи** — покупают тарифы, платят только за успешные
3. **Stripe webhook** — обязательно проверять подпись (security!)
4. **Тесты** — ни в коем случае не должны тратить посты
5. **Баланс** — хранится в JSON, обновляется в Railway

## 🤔 Вопросы и ответы

**Q: Почему Stripe webhook?**  
A: Это самый надежный способ получить уведомление о платеже. Другие варианты менее надежны.

**Q: Зачем удалять тестовые сообщения?**  
A: Чтобы группы не засорялись спам-сообщениями. Тесты должны быть чистыми.

**Q: Почему cooldown 30 сек на тесты?**  
A: Защита от спама. 5 тестов в день × 30 сек минимум = защита бота.

**Q: Может ли OWNER_ID тестировать бесконечно?**  
A: Да, это разработчик. Ему нужна свобода для отладки.
