## 🎯 ФАЗА 1: Система баланса постов в JSON + UI отображения

**Статус:** ✅ План готов  
**Файл:** [01_phase_balance_system.md](01_phase_balance_system.md)

### Контекст
Введение системы отслеживания баланса постов для каждого пользователя. Бесплатно 30 постов при первом подключении.

### Что делается
- Добавить поле `balance` в `broadcast_state.json`
- Меню "💰 Баланс постов" с отображением текущего баланса
- Меню выбора пакетов (3 варианта):
  - 🟢 100 постов / €3.99 (€0.0399/пост)
  - 🔵 300 постов / €7.99 (€0.0266/пост) — 33% дешевле ⭐ ОСНОВНОЙ
  - 🔴 1500 постов / €33.99 (€0.0226/пост) — 73% дешевле (якорь)
- Инициализация 30 бесплатных постов при /start
- История изменений баланса в JSON

### Визуал
```
💰 МОЙ БАЛАНС

🟢 Доступно: 600 постов
📊 Всего куплено: 600
📤 Всего потрачено: 0

[Купить тариф] [История] [Назад]
```

### Файлы для изменения
1. `bot/broadcast_manager.py` — добавить методы
2. `bot/main.py` — handlers для меню баланса

### Результат
- ✅ Пользователь видит баланс (30 постов)
- ✅ Есть UI для выбора пакета
- ✅ История ведется в JSON

## 📝 ФАЗА 2: Stripe интеграция + обработка платежей

**Статус:** ✅ План готов  
**Файл:** [02_phase_stripe_integration.md](02_phase_stripe_integration.md)

### Контекст
Подключение реального платежного сервиса Stripe для продажи пакетов.

### Что делается
- Создать в Stripe: 1 продукт + 3 цены (€3.99 / €7.99 / €33.99)
- Создать webhook endpoint: `/stripe-webhook`
- Обработка платежей: `stripe.Webhook.construct_event()` с проверкой подписи
- На успех: `bm.set_balance(old_balance + posts)`

### Stripe Constants
```python
STRIPE_PRICES = {
    "small": {"price_id": "price_...", "posts": 100, "price_eur": 3.99},
    "medium": {"price_id": "price_...", "posts": 300, "price_eur": 7.99},
    "large": {"price_id": "price_...", "posts": 1500, "price_eur": 33.99},
}

# .env:
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
```

### Файлы для изменения
1. **Создать:** `bot/stripe_handler.py`
2. **Изменить:** `bot/main.py` (callbacks для кнопок покупки)
3. **Изменить:** `requirements.txt` (добавить stripe>=5.0.0)
4. **Изменить:** `bot/.env` (добавить ключи)

### Результат
- ✅ Реальные платежи через Stripe
- ✅ Баланс обновляется через webhook
- ✅ Success/cancel страницы

## 🔍 ФАЗА 3: Проверка баланса перед рассылкой

**Статус:** ✅ План готов  
**Файл:** [03_phase_balance_check.md](03_phase_balance_check.md)

### Контекст
Перед "Массовой рассылкой" проверяем что хватает постов. Тест НЕ тратит посты.

### Что делается
- Проверка: `balance >= len(selected_groups)`
- Если не хватает → показать кнопку покупки
- Меню подтверждения: показать сколько потратится
- Тест передаётся флаг `is_test=True` → не тратит посты

### Визуал (недостаточно баланса)
```
❌ НЕДОСТАТОЧНО БАЛАНСА

Требуется: 10 постов
Доступно: 5 постов

[💳 Купить тариф] [◀️ Назад]
```

### Файлы для изменения
1. `bot/main.py` — broadcast_mass(), broadcast_test_v2()
2. `parser/broadcast_sender.py` — параметр `is_test`

### Результат
- ✅ Без баланса = рассылка невозможна
- ✅ Тест всегда бесплатный
- ✅ Пользователь видит затраты

## 🛡 ФАЗА 7: Защита от злоупотребления (rate limiting, cooldown)

**Статус:** ✅ План готов  
**Файл:** [07_phase_abuse_protection.md](07_phase_abuse_protection.md)

### Контекст
Защита от спама тестов: cooldown, дневной лимит, логирование. OWNER_ID освобождён.

### Что делается
- Cooldown: 30 сек между тестами
- Лимит: 5 тестов в день (регулярные пользователи)
- OWNER_ID: может тестировать бесконечно
- Логирование: подозрительная активность

### Логика
```python
def can_run_test(self, user_id: int) -> tuple[bool, str]:
    # OWNER может всегда
    if is_owner(user_id):
        return True, ""
    
    # Проверить cooldown (30 сек)
    if elapsed < 30:
        return False, f"Подождите {wait_seconds} сек"
    
    # Проверить дневной лимит (5 тестов)
    if daily_tests >= 5:
        return False, f"Уже {daily_tests} тестов сегодня"
    
    return True, ""
```

### Визуал при cooldown
```
🧪 ТЕСТ

⏳ Подождите 30 секунд перед следующим
(повтор в 08:45)

[Ладно]
```

### Файлы для изменения
1. `bot/broadcast_manager.py` — can_run_test(), record_test_run()
2. `bot/main.py` — broadcast_test_v2() (добавить проверку)

### Результат
- ✅ Защита от спама тестов
- ✅ OWNER_ID без ограничений
- ✅ Логирование активности
